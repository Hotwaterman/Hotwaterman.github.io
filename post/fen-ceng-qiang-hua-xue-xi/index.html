分层强化学习

本质上，为了选择适当的层次结构框架，需要知道域知识的可用程度（三者的组合也是可以想象的）：

  • 如果行为是完全指定的：选项框架
  • 如果行为被部分指定：HAM 框架
  • 如果较少的域知识可用：MAXQ、学习选项框架

在下面的表中，列方向分别表示：

  • 时间抽象：允许表示在不同的时间尺度上发生的动作方案的知识。我们讨论的是时间延长的动作。
  • 状态抽象：当子任务忽略环境状态的某些方面时，它要求以终止语句的形式指定子任务，而不是使用选项或部分策略方法。
  • “子任务：程序员提供的固定策略”意味着给定一组选项，系统通过这些选项学习策略。
  • “子任务：非确定性有限状态控制器”意味着给定部分策略的层次结构，系统学习整个问题的策略。
  • “子任务：终止语句和局部奖励函数”意味着给定一组子任务，系统学习整个问题的策略。
  • 分层最优策略是一种策略，它在给定层次结构的所有可表达的策略中是最优的。
  • 递归最优策略是指每个 SMDP（半马尔科夫决策过程）的策略最优，对应于分解中的每个子任务。

从根本上讲，HRL 的优势在于：通过解决扩展问题而更快的学习、通过减少维数灾难解决大的状态 - 动作空间问题的强大能力、在不同的抽象状态任务上使用子目标和抽象的动作，使用多层次的时间抽象，更真实和更好的泛化能力，从之前的任务中迁移知识，这些似乎已经触手可及，但还没有完全实现。

有很多很棒的想法和算法，但到目前为止还没有产生很大的影响，或广泛采用。坦率地说，RL 社区仍然存在合理的怀疑主义。一方面，例如通过更新层次代理的级别所产生的非平稳性等挑战，需要在实现上做出更多的努力，还需要引入额外的超参数。另一方面，我们还远没有达到合理的样本效率，正如 Fruit 等人在论文《Regret Minimization in MDPs with Options without Prior Knowledge》中所指出的：在选项框架中，选项是原子相互独立的宏动作。因此，我们失去了考虑选项之间的相互作用的潜在好处，而它可以显著提高样本效率。此外，如 OpenAI 博客文章中所述：“RL 研究者（包括我们自己）普遍认为，**长期的发展还需要本质上的进展，如分层强化学习。**我们的结果表明，我们没有给予如今的算法合适的评价——至少在他们有足够的规模和合理的探索方式的情况下。

除非 HRL 框架具有更强的用户友好性和更好的技术规范，HRL 将不会成为 RL 问题的标准。

也就是说，这些缺陷是与 HRL 的主要研究方向一致的已知问题，包括但不限于：

  • 高阶状态迁移函数非平稳性的更好管理
  • 自动学习层次结构
  • 更有效的探索
  • 丰富稀疏奖赏环境下有效分解的信号
  • 提高稳定性

我们已经发现深度网络中出现的分层结构可应用于感知任务，如计算机视觉中的卷积神经网络，但这些是感知层面上的分层。如果我们可以以同样的方式在 RL 中构建层次化的时间和行为系统，这将像卷积神经网络一样产生重大影响。目前仍有许多工作要做，并且还需要一个坚实的基础。

Semi-Markov decision process

In the MDP, after selecting an action, the agent will immediately jump to the next state according to the state transition equation ppp. In the semi-Markov decision process (SMDP), the number of steps from the current state to the next state is a random variable τ\tauτ, that is, after selecting an action aaa in a certain state sss, it will transfer to the next state s′s′s′ with a probability after τ\tauτ steps. The state transition probability at this time is the joint probability p(s′,τ∣s,a)p(s′,\tau\mid s, a)p(s′,τ∣s,a) of sss and τ\tauτ. When the value of τ\tauτ is a real value, SMDP constructs a continuous-time-discrete-event system; when the value of τ\tauτ is a positive integer, it is a discrete-time SMDP. For the sake of simplicity, most HRL problems are discussed on discrete-time SMDP.

We consider a finite-horizon, goal-conditioned MDP defined as a tuple (S,G,A,P,R,γ)(\mathcal{S,G,A,P,R},\gamma)(S,G,A,P,R,γ), where S\mathcal{S}S is a state set, G\mathcal{G}G is a goal set, A\mathcal{A}A is an action set, P:S×A×S→R\mathcal{P:S\times A \times S}\to \mathbb{R}P:S×A×S→R is a state transition, R:S×A→R\mathcal{R:S\times A}\to \mathbb{R}R:S×A→R is a reward function, and γ\gammaγ is a discount factor.

Hindsight Experience Replay

Because the reward is sparse, it will inevitably lead to many complete transitions that cannot reach the preset goal, obviously, these trajectories cannot be used for the training of the preset goal. For example, if there is a transition whose last step does not reach the preset goal, then this step will not get a positive reward, but if the goal is changed to the state that has been reached by the last step, then this step will get a positive reward. Beyond this step, the goal of the entire transition should be modified to this state, which is the meaning of hindsight. The reason for this is that no matter what the goal setting is, it will not affect the system dynamics. Taking the same action in the same state will reach the same next state, so we can change the generated trajectory with the goal reset to any value. In this way, the number of successful samples in the experience memory will be expanded.

Off-policy correction

It is straightforward to see that in HRL, the training of high-level policy depends on the low-level policy, that is, the reward and arrival state in the high-level transition tuple depend on the policy adopted by the low-level controller, which results in the previous transition tuple cannot be used for subsequent training. This problem may cause the high/low level non-stationary under off-policy. To solve this problem, HIRO proposes an off-policy correction method. The high-level action gtg_tgt​ which in the past induced a low-level behavior at:t+c−1∼μlo(st:t+c−1,gt:t+c−1)a_t:t+c−1\thicksim \mu^{lo}(s_{t:t+c−1},g_{t:t+c−1})at​:t+c−1∼μlo(st:t+c−1​,gt:t+c−1​) may be re-labeled to a goal g~t\widetilde{g}_tg​t​ which is likely to induce the same low-level behavior with the current lower-level policy. HAC proposes another method to solve the non-stationary brought by different levels of training, that is, using the idea of hindsight to assume that the low-level policy has reached the optimal. HAC proposes three transitions: (1) {hindsight action transition}. For the non-lowest level, the action space is the state space. If the goal set by the upper level is not reached, the reached state will be regarded as the goal, which is equivalent to the higher-level action of hindsight is done so that the higher-level is trained based on the "optimal" lower-level policy, to temporarily ignore the change of the lower-level policy. (2){hindsight goal transition}. This is what the original HER did. Adjust the goal to solve the problem of sparse reward; (3){subgoal testing transition}. Under the above two adjustments, the transition used by the agent to update is some reachable goals. It may not be achievable for some distant goals. Therefore, a certain probability is used to test whether the lower-level policy can achieve the current goal. When the goal cannot be completed by the lower-level, a penalty reward will be given.

Goal representation

FuN uses as an intrinsic reward the direction taken in a feature space. The features used are those which are useful to the task, they are therefore inherently dependent of the extrinsic reward.

Then, RIG proposes to build the feature space independently from the task with variationally auto-encoder (VAE); but this approach can be very sensitive to distractors (i.e. useless features for the task or goal, inside states) and does not allow to correctly weight features.

Skew-fit recently improves over RIG by weighting rare states, leading to more diverse policies. Both RIG and Skew-fit are efficient in reaching a given state.

The current state-of-art method, Sub-optimal representation learning, bounds sub-optimality of the goal representation with respect to the task, giving theoretical guarantees; the agent can solve the task with a hierarchical policy as precisely as with a simple low-level policy.

Goal generation with adversarial training

In the paradigm of adversarial training, two modules face each other: the first one, the generator, tries to fool the second, the discriminator, which must avoid to be mistaken.

GoalGAN learns to generate increasingly complex continuous goals with a Generative Adversarial Networks (GAN) in order to make the policy progressively learn to go everywhere.

The generator of the GAN learns to produce goals of intermediate difficulty and the discriminator learns to distinguish these goals from others.

Intermediate difficulty is characterized as a goal that an agent achieves from time to time.

In this article, the intrinsic reward relies on an hand-engineered indicator function which attributes a binary reward if the agent is close to a goal.

Similar issue can be found in Self-play HRL.In this article, adversarial method is applied to learn a goal space which is a compressed state space.

During a pre-training step, an agent (generator) tries to go into the state that an other agent (discriminator) went into, whereas the discriminator learns to go into areas the generator cannot reach.

In other words, the generator tries to produce trajectories that the discriminator can not differentiate from its own.

Reverse form of the mutual information

where log⁡qϕ(z∣s)\log q_{\phi}(z\mid s)logqϕ​(z∣s) is fitted by maximum likelihood on (s,z)(s,z)(s,z) tuple collected by deploying the policy in the environment.

\begin{equation} \begin{aligned} \mathcal{F}(\theta)&\triangleq I(S;Z)+\mathcal{H}[A\mid S]-I(A;Z\mid S) \\ &= \mathcal{H}[Z]-\mathcal{H}[Z\mid S]+\mathcal{H}[A\mid S,Z] \label{eq14} \end{aligned} \end{equation}

Challenge

Through the previous introduction, although HRL has many advantages, it is not difficult to see that there are still many problems. Several questions still remain to be answered:

Better management of the non-stationarity of the higher-level state transition functions

Although both HAC and HIRO use the re-label experience method to solve the non-stationary problem between different levels to a certain extent, the HIRO method is difficult to calculate and the convergence speed is full. HAC will cause the meta controller to tend to choose A more conservative goal. How to solve this changing MDP control problem and solve the non-stationary problem is still an urgent problem in HRL.

Learn hierarchies automatically}

It is not difficult to see from some previous work that HRL still needs to manually design a large number of hyper-parameters. At the same time, a large amount of HRL work is concentrated on a two-layer control structure, and only individual papers discuss the possibility of three or four levels. However, the number of structural levels or the number of steps still cannot achieve automatic learning.

More efficient exploration

One of the reasons why HRL can perform better than ordinary reinforcement learning is because the choice of goals can drive the agent to take more efficient and meaningful exploration.

Improve stability

HRL is more difficult to train than general reinforcement learning methods because of its structural complexity and the new instability introduced between levels.

At the same time, there are many parameters in HRL algorithm, which makes it more difficult to reproduce and apply. This is also a problem that needs to be considered.

Conclusion

This article mainly introduces the recent research on hierarchical deep reinforcement learning, and divides the current mainstream hierarchical frameworks into goal-based framework and temporal abstraction based framework. After analyzing the advantages and disadvantages of their respective mainstream methods, we found that HRL has superiority over ordinary reinforcement learning methods in solving complex problems and transferring skills, but because of its structural complexity and instability, more research and improvement are still needed.
