Meta Q learning

2020-06-25 7 min read

MQL提出来三个idea：1）如果可以访问上下文变量的话，Q learning比meta RL更有竞争力。2）一个最大化训练任务间的平均奖励的多任务目标是一种对RL策略进行meta-train的有效方法。3）元训练中重播缓冲区的过去数据可以利用off-policy更新来适应新环境中的策略。MQL借鉴了倾向估计中的思想，从而扩大了可用于adaptation的数据量。

meta-RL的效果如何呢？图一表明了两种原型meta-RL在四个标准连续控制基准上的表现。baseline为一种最大化训练任务间的平均奖励和使用代表轨迹的“context variable”的off-policy RL算法（TD3）。

提出了一种off-policy的meta-RL叫MQL：利用off-policy更新最大化所有meta-training任务的平均奖励。

designing context

在meta-RL中的任务特性可以被看作是部分可见MDP的隐藏变量，我们设计了一种依赖于{(xi,ui,ri)}i≤t\{(x_i,u_i,r_i)\}_{i\le t}{(xi​,ui​,ri​)}i≤t​ 的循环上下文变量context variable ztz_tzt​ ，将其设为门控循环单元（GRU）在时间t的隐藏状态。所有的策略和价值函数都取决于上下文，uθ(x,z)u_{\theta}(x,z)uθ​(x,z), qφ(x,u,z)q_{\varphi}(x,u,z)qφ​(x,u,z) 。任何其他的递归模型可以用来设计上下文，使用GRU是因为它在丰富表现和复杂计算之间有个很好的折中。

**remark 2 ** （MQL使用的确定性上下文不是置换不变的），我们试图在设计context时进行简化，使用像GRU这样现成的模型并且不是permutation invariant。时间的方向向agent提供了关于任务的动态信息，向前运动和向后运动虽然有相同的状态轨迹，但是顺序不同。MQL的context是轨迹的确定性函数。
