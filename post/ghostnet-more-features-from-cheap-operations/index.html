GhostNet: More Features from Cheap Operations

2020-07-30 8 min read

CNNs are very powerful, especially the emergence of resnet, which makes it possible for network performance to become better and better as the depth of the network becomes deeper, but this leads to a large and slow model. Recently, various researches have appeared in the design of lightweight networks, such as model pruning, quantification, knowledge distillation, or some simplified network structure design. Today I am going to introduce a paper on CVPR2020 from Huawei Noah Lab (I think this is designed for Huawei mobile phones). Ghostnet can achieve the same accuracy as SOTA algorithm with less computational complexity .

The paper provides a new Ghost module, which aims to generate more feature maps through cheap operations. Based on a set of original feature maps, the author applies a series of linear transformations to generate many "Ghost feature maps" that can extract the required information from the original features at a small cost. The Ghost module is plug-and-play, and Ghost bottleneck is obtained by stacking Ghost modules, and then a lightweight neural network-GhostNet is built.

A well-trained deep neural network usually contains rich or even redundant feature maps to ensure a comprehensive understanding of the input data. As shown in the figure above, in ResNet-50, the feature map after the first residual block processing is taken out, and the three similar feature maps are annotated with the same color box for the example. One feature map in the pair can be obtained by transforming the other feature map through a cheap operation (represented by a wrench), and one of the feature maps can be regarded as a "ghost" of the other. Because, this article proposes that not all feature maps need to be obtained by convolution operation, "ghost" feature maps can be generated by cheaper operations.

Ghost Module

Although recent work such as MobileNet and ShuffleNet introduced deep convolution or shuffling operations to use smaller convolution kernels (floating-number operations) to build effective CNNs, the remaining 1×11\times 11×1 convolutional layer will still occupy A lot of memory and FLOPs.

The author proposes to reduce the required resources, namely the convolution kernels used to generate them. According to the above formula, the number of parameters to be optimized (parameters in fff and bbb) is determined by the size of the input and output feature maps. As observed in Figure 1, the output feature map of the convolutional layer usually contains a lot of redundancy, and some of them may be similar to each other. The author pointed out that it is not necessary to use a large number of FLOPs and parameters to generate these redundant feature maps one by one. Instead, the output feature maps are "ghosts" of a few original feature maps through some cheap conversion. These original feature maps usually have a small size and are generated by ordinary convolution kernels.
